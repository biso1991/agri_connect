
# Django REST Framework: Serializers and View Handling

## **1. Overview of Serializers**

Serializers in Django REST Framework (DRF) help in:

* Converting complex data types (e.g., Django models) into JSON for responses.
* Validating and deserializing JSON into Python objects.

### **Basic Serializer API:**

* `.data`: Serialized data ready for response.
* `.is_valid()`: Validates input data.
* `.validated_data`: Access validated input data.
* `.errors`: Shows validation errors.
* `.save()`: Creates or updates an object instance.

## **5. Request Handling**

* **File Uploads:**
  * Use `request.FILES` for files.
  * Use `request.data` for non-file data.
* **Query Parameters:**
  * Access GET parameters via `request.query_params`.

---

## **6. Additional Notes**

* **APIView:** Base class for views with basic request handling.
* **GenericAPIView:** Adds queryset, serializer, and filtering support.
* **Mixins:** Modular behavior (e.g., create, retrieve, update, delete).
* **Viewsets:** Combines logic and routing for CRUD operations.

For more details, explore the [Classy DRF](https://www.cdrf.co/) reference.
